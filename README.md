# 강동어울림복지관 "이용상담문의" 모니터링 및 구글챗 알림 시스템

이 프로젝트는 강동어울림복지관 홈페이지의 **"이용상담문의"** 게시판을 실시간으로 감지하고, 새 글이 등록되었을 때 구글 챗(Google Chat)으로 자동 알림을 전송하는 모니터링 시스템입니다.

---

## 🌟 주요 기능

1. **실시간 모니터링**: 5분 주기로 복지관 게시판 목록의 HTML을 크롤링하여 신규 게시글을 감지합니다.
2. **시간대별 발송 정책**:
   - **평일 09:00 ~ 18:00 (KST)**: 새 문의글 감지 시 즉시 구글챗으로 알림을 발송합니다.
   - **그 외 시간대 및 주말**: 감지된 게시글을 데이터베이스(Queue)에 적재하고, **다음 평일 오전 9시 정각**에 요약 카드로 일괄 발송합니다.
3. **관리자 대시보드 (GitHub Pages)**:
   - 글래스모피즘(Glassmorphism) 기반의 세련된 모던 디자인.
   - 모니터링할 복지관 게시판 URL 목록 관리(추가, 수정, 삭제, 활성화 토글).
   - 알림을 발송할 구글챗 웹훅(Incoming Webhook) 관리 및 즉시 테스트 발송 기능.
   - 최근 감지된 게시글 로그 및 알림 발송 이력 조회.
   - 비밀번호 보안 해시 저장 및 세션 만료 관리.

## 🛠️ 기술 아키텍처 및 폴더 구조

```
├── index.html           # 로그인 화면 (클라이언트 세션 제어)
├── dashboard.html      # 관리자 대시보드 메인 화면
├── assets/
│   ├── css/
│   │   └── style.css   # 글래스모피즘 디자인 시스템 및 Paperlogy 폰트
│   └── js/
│       ├── api.js      # CORS preflight를 회피하는 GAS API 통신 모듈
│       ├── auth.js     # 관리자 로그인 인증 및 세션 검증
│       └── dashboard.js# CRUD, 로그 출력, 비밀번호 제어 비즈니스 로직
├── backend/
│   └── Code.gs         # Google Apps Script 백엔드 소스코드
└── README.md           # 본 안내 가이드
```

- **프론트엔드**: GitHub Pages를 통한 정적 배포 (Vanilla JS, 빌드 도구 불필요).
- **백엔드**: Google Apps Script(GAS)를 Web App(REST API)으로 구동.
- **데이터베이스**: Google Sheets를 이용해 데이터 적재.
- **스케줄러**: Google Apps Script의 내장 **시간 기반 트리거(Time-driven Trigger)** 사용.

## 🚀 설정 및 배포 방법

### 1단계: Google Sheets 및 Apps Script 설정 (백엔드)
1. **Google Sheets 생성**:
   - 새 구글 시트(Google Spreadsheet)를 생성합니다.
   - 시트의 URL에서 스프레드시트 ID(예: `https://docs.google.com/spreadsheets/d/스프레드시트ID/edit`)를 기록해 둡니다.
2. **Apps Script 에디터 열기**:
   - 시트 상단 메뉴에서 **확장 프로그램** -> **Apps Script**를 클릭합니다.
3. **코드 붙여넣기**:
   - 에디터 안의 기본 코드를 모두 지우고, 이 리포지토리의 [Code.gs](backend/Code.gs) 파일의 전체 내용을 복사하여 붙여넣습니다.
4. **시트 ID 설정 (선택 사항)**:
   - 스크립트 에디터 좌측의 **설정(톱니바퀴 아이콘)** -> **스크립트 속성**에 `SPREADSHEET_ID` 항목을 추가하고 구글 시트 ID 값을 넣습니다. (시트에 바인딩된 경우 생략해도 자동으로 연결됩니다.)
5. **웹앱 배포**:
   - 에디터 우측 상단의 **배포** -> **새 배포**를 클릭합니다.
   - 유형 선택에서 **웹앱**을 클릭합니다.
   - 설정:
     - **설명**: `v1.0.0`
     - **웹앱을 실행할 사용자**: `나 (본인의 구글 계정)`
     - **액세스 권한이 있는 사용자**: `모든 사용자 (Anyone)`
   - **배포** 버튼을 누르고 구글 계정 액세스 승인 절차를 진행합니다.
   - 배포 완료 화면에 나타나는 **웹앱 URL**(`https://script.google.com/macros/s/.../exec`)을 복사해 둡니다.

### 2단계: 스케줄러 트리거 설정
실시간으로 복지관 글을 긁어오도록 스케줄러를 등록합니다.
1. Apps Script 에디터 좌측 메뉴에서 **트리거 (시계 아이콘)**를 클릭합니다.
2. 우측 하단의 **트리거 추가** 버튼을 누릅니다.
3. 다음과 같이 설정합니다:
   - **실행할 함수 선택**: `checkNewPosts`
   - **실행할 배포 버전 선택**: `기기 기본값` 또는 `Head`
   - **이벤트 소스 선택**: `시간 기반`
   - **시간 기반 트리거 유형 선택**: `분 단위 타이머`
   - **시간 간격 선택(분)**: `1분마다` (실시간 모니터링 강화를 위해 1분으로 설정)
4. **저장**을 누릅니다.

### 3단계: 프론트엔드 웹앱 연동 및 배포
1. **API URL 수정**:
   - 리포지토리의 `assets/js/api.js` 파일을 열고, 최상단에 있는 `API_BASE` 상수에 복사해 둔 **웹앱 URL**을 붙여넣습니다.
   ```javascript
   const API_BASE = "https://script.google.com/macros/s/실제_배포된_GAS_ID/exec";
   ```
2. **GitHub Pages 배포**:
   - 코드를 GitHub 저장소에 푸시합니다.
   - GitHub 저장소 웹 페이지에서 **Settings** -> **Pages** 탭으로 이동합니다.
   - **Build and deployment** 항목의 **Branch**를 `main`으로 설정하고 폴더를 `/(root)`로 지정한 뒤 **Save**를 누릅니다.
   - 수 분 내에 GitHub Pages 주소(예: `https://사용자명.github.io/boardalarm/`)로 배포됩니다.

## 🔒 보안 및 초기 비밀번호

- **초기 접속 계정**:
  - **아이디**: `admin`
  - **비밀번호**: `1234`
- **비밀번호 해시 저장**: 비밀번호는 브라우저나 소스코드에 하드코딩되지 않으며, SHA-256 해시 알고리즘으로 암호화되어 Google Sheets Settings 및 Script Properties에 안전하게 저장됩니다.
- **보안 권장사항**: 최초 로그인 후 반드시 대시보드의 **시스템 설정** 탭에서 비밀번호를 새롭게 변경해 주시기 바랍니다.
- **웹훅 URL 보안**: 등록한 Incoming Webhook URL은 대시보드 화면상에서 전체 주소가 마스킹 처리되며, 실제 발송 로직은 안전하게 구글 서버 백엔드 내에서만 호출됩니다.
